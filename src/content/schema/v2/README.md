# Story Schema v2

`schema v2` хранит сюжет вне монолитного `storyContent.js` и делит данные по зонам ответственности.

## Структура

- `meta.json` — версия контента, стартовая сцена, пролог.
- `apps.json` — приложения рабочего стола.
- `acts.json` — акты и их краткое назначение.
- `chats.json` — список чатов мессенджера.
- `appData.json` — наполнение заметок, календаря, терминала и конвейера.
- `assets.json` — ассеты и их смысл.
- `evidence.json` — улики и сюжетные артефакты.
- `sequences.json` — системные последовательности интерфейса.
- `scenes/*.json` — сцены акта, сгруппированные по файлам.

## Правила

- Любая сущность имеет стабильный `id`.
- Чатовые сцены ссылаются на чат через `chatId`.
- Переходы между сценами задаются через `nextSceneId` или будут расширены до явного `goto`.
- Все новые механики, которые рантайм пока не умеет, кладутся в поле `schema`, чтобы не ломать совместимость.
- `storySchemaV2.js` собирает все файлы в один объект и проецирует его в текущий `STORY_CONTENT`.

## Текущий статус
